Windows API 学习记录

Posted on 2024-03-13 | In Reverse

进程启动

WinExec

函数在winbase.h中声明。

  • lpCmdLine：命令行参数
  • uCmdShow：显示选项，在winuser.h中定义，重点关注：
    • SW_HIDE：隐藏窗口。
    • SW_SHOWNORMAL：激活并显示窗口。
    • SW_SHOW：激活窗口并以当前大小和位置显示窗口。

ShellExecute

函数在shellapi.h中声明。

  • hwnd：用于显示 UI 或错误消息的父窗口的句柄。如果操作未与窗口关联，则此值可以为NULL 。
  • lpOperation：用于指定要进行的操作。
    • edit：启动编辑器并打开文档进行编辑。
    • explore：寻找lpFile指定的文件夹。
    • find：从lpDirectory指定的目录开始启动搜索。
    • open：打开lpFile参数指定的项目。
    • NULL：默认为open操作。
  • lpFile：执行指定动词的文件或对象。
  • lpParameters：指定要传递给应用程序的参数。如果lpFile为文件应为NULL。
  • nShowCmd：同WinExec中uCmdShow的用法。

CreateProcess

函数在processthreadsapi.h中声明。

  • lpApplicationName：要执行的模块的名称。为NULL则是lpCommandLine中第一个空格分隔的标记。
  • lpCommandLine：要执行的命令行。
  • lpProcessAttributes：指向 SECURITY_ATTRIBUTES 结构（确定子进程是否可以继承新进程对象的返回句柄）的指针。可以为NULL。
  • lpThreadAttributes：基本同lpProcessAttributes。
  • bInheritHandles：TRUE则调用进程中的每个可继承句柄都由新进程继承。
  • dwCreationFlags：控制优先级类和进程的创建的标志。
  • lpEnvironment：指向新进程的环境块的指针。 如果此参数为NULL，则新进程将使用调用进程的环境。
  • lpCurrentDirectory：进程当前目录的完整路径。如果此参数为 NULL，则新进程将具有与调用进程相同的当前驱动器和目录。
  • lpStartupInfo：指向 STARTUPINFO 或 STARTUPINFOEX 结构的指针。
  • lpProcessInformation：指向接收有关新进程的标识信息的 PROCESS_INFORMATION 结构的指针。

进程访问

OpenProcess

函数在processthreadsapi.h中声明。打开现有的本地进程对象。

  • dwDesiredAccess：对进程对象的访问。根据进程的安全描述符检查此访问权限。
    • PROCESS_ALL_ACCESS：进程对象的所有可能的访问权限。
    • PROCESS_SUSPEND_RESUME：暂停或恢复进程所必需的。
    • PROCESS_VM_OPERATION：需要对进程的地址空间执行操作。
    • PROCESS_VM_READ：使用 ReadProcessMemory 读取进程中的内存所必需的。
    • PROCESS_VM_WRITE：使用 WriteProcessMemory 写入进程中的内存。
    • PROCESS_TERMINATE：使用 TerminateProcess 终止进程所必需的。
  • bInheritHandle：如果该值为 TRUE，则该进程创建的进程将继承该句柄。
  • dwProcessId：要打开的本地进程的标识符。

如果函数失败，则返回值为 NULL。

TerminateProcess

函数在processthreadsapi.h中声明。终止指定的进程及其所有线程。

  • hProcess：要终止的进程句柄。句柄必须具有 PROCESS_TERMINATE 访问权限。
  • uExitCode：进程和线程将使用的退出代码，由于此调用而终止。

进程遍历

CreateToolhelp32Snapshot

函数在tlhelp32.h中声明。获取指定进程以及这些进程使用的堆、模块和线程的快照。

  • dwFlags：要包含在快照中的系统部分。
    • TH32CS_INHERIT：指示快照句柄是可继承的。
    • TH32CS_SNAPALL：包括系统中的所有进程和线程，以及 th32ProcessID 中指定的进程的堆和模块。
    • TH32CS_SNAPPROCESS：包括系统中快照中的所有进程。
    • TH32CS_SNAPTHREAD：包括快照系统中的所有线程。
  • th32ProcessID：要包含在快照中的进程的进程标识符。此参数可以为零以指示当前进程。

如果函数失败，它将返回 INVALID_HANDLE_VALUE。

Process32First

函数在tlhelp32.h中声明。检索有关系统快照中遇到的第一个进程的信息。

  • hSnapshot：快照的句柄，该句柄是从上次调用 CreateToolhelp32Snapshot 函数返回的。
  • lppe：指向 PROCESSENTRY32 结构的指针。

内存申请

VirtualAlloc

函数在memoryapi.h中声明。保留、提交或更改调用进程的虚拟地址空间中页面区域的状态。 此函数分配的内存会自动初始化为零。

  • lpAddress：要分配的区域的起始地址。如果此参数为 NULL，则系统会确定分配区域的位置。
  • dwSize：区域的大小（以字节为单位）。
  • flAllocationType：内存分配的类型。
    • MEM_COMMIT：提交指定的内存区域，分配实际的物理存储。
    • MEM_RESERVE：保留指定的内存区域，无需分配实际的物理存储。
  • flProtect：要分配的页区域的内存保护。
    • PAGE_NOACCESS：禁用对已提交页面区域的所有访问。
    • PAGE_EXECUTE：启用对页面已提交区域的执行访问。
    • PAGE_EXECUTE_READ：启用对页面已提交区域的执行或只读访问。
    • PAGE_EXECUTE_READWRITE：启用对已提交页面区域的执行、只读或读/写访问权限。

VirtualAlloc2

函数在memoryapi.h中声明。保留、提交或更改指定进程的虚拟地址空间中内存区域的状态， (分配的内存初始化为零) 。

  • Process：进程的句柄。句柄必须具有 PROCESS_VM_OPERATION 访问权限。 如果 Process 为 NULL，则该函数为调用进程分配内存。
  • BaseAddress：为要分配的页面区域指定所需起始地址的指针。如果 BaseAddress 为 NULL，则该函数确定区域分配位置。
  • ExtendedParameters：指向 MEM_EXTENDED_PARAMETER 类型的一个或多个扩展参数的可选指针。
  • ParameterCount：ExtendedParameters 指向的扩展参数数。

VirtualAllocEx

函数在memoryapi.h中声明。在指定进程的虚拟地址空间中保留、提交或更改内存区域的状态。 函数将它分配的内存初始化为零。

  • Process：进程的句柄。句柄必须具有 PROCESS_VM_OPERATION 访问权限。

其他参数上述都给出。

HeapAlloc

函数在heapapi.h中声明。从堆中分配内存块。 分配的内存不可移动。

  • hHeap：要从中分配内存的堆的句柄。 此句柄由 HeapCreate 或 GetProcessHeap 函数返回。
  • dwFlags：堆分配选项。
    • HEAP_ZERO_MEMORY：分配的内存将初始化为零。
  • dwBytes：要分配的字节数。

解析相关的 HeapCreate 或 GetProcessHeap 函数。

HeapCreate 函数在heapapi.h中声明。创建可由调用进程使用的专用堆对象。

  • flOptions：堆分配选项。 此参数可以是 0 或以下一个或多个值。
    • HEAP_GENERATE_EXCEPTIONS：系统将引发异常以指示函数失败。
    • HEAP_NO_SERIALIZE：序列化访问将不用于此分配。
    • HEAP_CREATE_ENABLE_EXECUTE：堆分配的所有内存块都允许代码执行。
  • dwInitialSize：堆的初始大小（以字节为单位）。
  • dwMaximumSize：堆的最大大小（以字节为单位）。如果 dwMaximumSize 为 0，堆大小可能会增大。

GetProcessHeap 函数在heapapi.h中声明。检索调用进程的默认堆的句柄。

GlobalAlloc

函数在winbase.h中声明。从堆中分配指定数量的字节。

  • uFlags：内存分配属性。如果指定 0，则默认值为GMEM_FIXED。
    • GMEM_FIXED：分配固定内存。返回值是一个指针。
    • GMEM_MOVEABLE：分配可移动内存。内存块不会在物理内存内移动，但可以在默认堆内移动。
    • GMEM_ZEROINIT：将内存内容初始化为0。
  • dwBytes：要分配的字节数。

CoTaskMemAlloc

函数在combaseapi.h中声明。分配内存块。

  • cb：要分配的内存块的大小，以字节为单位。

AllocADsMem

函数在adshlp.h中声明。分配指定大小的内存块。

  • cb：包含要分配的大小（以字节为单位）。

WriteProcessMemory

函数在memoryapi.h中声明。将数据写入指定进程中可访问的内存区域。

  • hProcess：要修改的进程内存的句柄。该句柄必须具有对进程的 PROCESS_VM_WRITE 和 PROCESS_VM_OPERATION 访问权限。
  • lpBaseAddress：指向写入数据的指定进程中的基地址的指针。
  • lpBuffer：指向缓冲区的指针，该缓冲区包含要写入指定进程的地址空间中的数据。
  • nSize：要写入指定进程的字节数。
  • lpNumberOfBytesWritten：指向变量的指针，该变量接收传输到指定进程的字节数。可为NULL。

内存修改

VirtualProtect

函数在memoryapi.h中声明。更改对调用进程的虚拟地址空间中已提交页面区域的保护。

  • lpAddress：要更改访问保护属性的页面区域的起始页面的地址。指定区域中的所有页面必须位于使用MEM_RESERVE调用VirtualAlloc或VirtualAllocEx函数时分配的同一保留区域内。
  • dwSize：要更改访问保护属性的区域的大小，以字节为单位。
  • flNewProtect：内存保护选项。
  • lpflOldProtect：指向变量的指针，该变量接收指定页面区域中第一页的先前访问保护值。

VirtualProtectEx

函数在memoryapi.h中声明。更改对指定进程的虚拟地址空间中已提交页面区域的保护。

  • hProcess：要更改其内存保护的进程的句柄。该句柄必须具有PROCESS_VM_OPERATION访问权限。

线程操作

CreateThread

函数在processthreadsapi.h中声明。创建一个线程以在调用进程的虚拟地址空间内执行。

  • lpThreadAttributes：指向SECURITY_ATTRIBUTES结构的指针 ，该结构确定返回的句柄是否可以由子进程继承。如果 lpThreadAttributes为NULL，则句柄不能被继承。
  • dwStackSize：堆栈的初始大小（以字节为单位）。如果此参数为零，则新线程使用可执行文件的默认大小。
  • lpStartAddress：指向要由线程执行的应用程序定义的函数的指针。
  • lpParameter：指向要传递给线程的变量的指针。
  • dwCreationFlags：控制线程创建的标志。
    • 0：线程创建后立即运行。
    • CREATE_SUSPENDED：该线程在挂起状态下创建，直到调用 ResumeThread函数 后才会运行 。
    • STACK_SIZE_PARAM_IS_A_RESERVATION：dwStackSize参数指定堆栈的初始保留大小。
  • lpThreadId：指向接收线程标识符的变量的指针。如果此参数为 NULL，则不返回线程标识符。

来看相关的 ResumeThread函数 ：

减少线程的挂起计数。当挂起计数减至零时，线程的执行将恢复。

  • hThread：要重新启动的线程的句柄。此句柄必须具有 THREAD_SUSPEND_RESUME 访问权限。

WaitForSingleObject

函数在synchapi.h中定义。等待指定对象处于有信号状态或超时间隔已过。

  • hHandle：对象的句柄。该句柄必须具有 SYNCHRONIZE 访问权限。
  • dwMilliseconds：超时间隔，以毫秒为单位。如果指定了非零值，则该函数将等待，直到对象收到信号或间隔过去。如果dwMilliseconds为零，则如果对象没有收到信号，函数不会进入等待状态；它总是立即返回。如果dwMilliseconds为INFINITE，则仅当对象收到信号时该函数才会返回。

CreateRemoteThread

函数在processthreadsapi.h中声明。创建一个在另一个进程的虚拟地址空间中运行的线程。

  • hProcess：要在其中创建线程的进程的句柄。该句柄必须具有PROCESS_CREATE_THREAD、PROCESS_QUERY_INFORMATION、PROCESS_VM_OPERATION、PROCESS_VM_WRITE和PROCESS_VM_READ访问权限。

注意需要先向远程进程注入代码，然后再创建远程线程执行。
