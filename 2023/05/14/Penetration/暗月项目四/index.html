初探多重网段的渗透

Posted on 2023-05-14 | In Penetration

总结

本次学习的是暗月项目四：初探多重网段的渗透。

网站测试：

  • new.cc123.com：学习了 dedecms 漏洞的利用；尝试了利用 aspxspy.aspx 木马进行权限的提升，利用 msf 中的 multi/handler 攻击模块进行 shell 的反弹；最后利用 msf 中的漏洞寻找模块 multi/recon/local_exploit_suggester 提升权限至系统权限。
  • ww2.cc123.com：学习了后台登录的一个 sql 注入漏洞；利用 sqlmap 侦测到数据库用户为超级用户，拿到了数据库的 shell；除此以外还测试了 kindeditor 的文件上传漏洞。

内网渗透：

学习了利用 metasploit 中的 mimikatz 来收集用户凭证；反向和正向连接远程服务器；利用 auxiliary/server/socks_proxy 和 proxychains 进行代理；利用 SocksCap64 使得主机也可以访问到内网。

最终也是跟着暗月佬的视频拿下了三台主机，虽然路途坎坷。

根据织梦的特点找到版本号。

Dedecms 20150618 注入漏洞(可注入管理员账户) - 开水网络 (hekaiyu.cn)

登陆后台管理系统，http://new.cc123.com/dede/，发现了可以上传木马文件，尝试用蚁剑连接，成功。但是无法成功执行命令，这也就是为什么我们没有办法反弹shell的原因。

这里根据网站目录发现可能支持 asp.net，而且相较于 php，.net的权限更加大，这里尝试上传 aspx 木马。

webshell/AspxSpy2014Final.aspx at master · tennc/webshell · GitHub

上传该木马，登陆密码为 admin，发现可以执行相关命令。

ww2.cc123.com

发现后台，通过抓包提交，发现存在验证码的重用，可以爆破密码。

除此以外测试一下 SQL注入，通过闭合用户名和注释，发现绕过了密码的判断，出现了 302 重定向。也就是说只判断了表中是否有 admin，如果有就进行跳转。

进入后台我们发现了可以添加或者是修改管理员。同时我们也发现了留言板的 xss。

除此以外，在新闻资讯列表中发现修改信息时存在 sql注入。

发现用户是sa，sa是SQL Server数据库中的系统管理员账户，具有最高权限。它可以访问和管理所有数据库，并且可以执行任何操作，包括创建、修改和删除数据库对象，更改数据库配置选项等。因此，sa账户通常被用作在SQL Server上运行的应用程序的默认管理员账户。

除上述以外，我们发现了 kindeditor 的一个漏洞。

大批量Kindeditor文件上传事件的漏洞分析 - FreeBuf网络安全行业门户

通过修改上传数据包中的 dir 字段为 file，我们就可以实现上传 html 文件。
