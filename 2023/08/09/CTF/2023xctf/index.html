首先是Add功能，会先malloc(0x2000)一个堆块，然后向里面写入内容，然后根据输入内容的长度再申请一个堆块，并将内容memcpy到新申请的堆块中，最后释放掉大堆块。

然后是Show功能，没什么好说的正常形态。接着是Edit功能，调用了和Add功能一样的函数。

fcalc

一道浮点数绕过的题目，栈可执行，复现的过程中遇到了两个问题。

问题一：程序逻辑和栈帧的理解，忽略了alloc函数一直没看懂栈的结构，alloc函数的作用是在栈上开辟一段空间出来，但是很有可能会造成溢出，目前是已经被弃用了。在这道题中，变量buf所指向的栈空间位于变量s所指向的栈空间高0x40字节的地方。

问题二：是对于浮点数的理解，通过调试发现需要写一个跳转指令跳到shellcode上，但是该跳转指令需要绕过浮点数的检查。

先复习一下浮点数：浮点数详解

IEEE754浮点数的结构：

利用特性：阶码（exp）全为0时，当尾数（frac）全为0时，根据数符（s），表示无穷大或者是无穷小；当尾数（frac）不全为0时，该浮点数表示非数值（NaN），而NaN具有以下的特点：

Comparison NaN ≥ x NaN ≤ x NaN > x NaN < x NaN = x NaN ≠ x
Result False False False False False True

所以只要保证我们跳转指令的高位为0x7ff或0xfff即可绕过检查。
