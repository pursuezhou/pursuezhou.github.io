Penetration
bugku

场景1

看页面源代码就能找到：

场景2

根据提示访问/admin试试看，找到登陆页面，尝试弱口令直接嗦admin/admin，拿到管理员权限。

场景3

尝试修改上传后缀上传木马，能够成功上传，但是似乎会将php代码注释成html返回给我们，失败。

在php教程发现将我们修改的代码上传成php文件，拿到shell和flag。

场景4

提示root和数据库，那flag应该是在数据库里，想办法拿到root的数据库密码，尝试找数据库配置文件。

场景5

看提示应该是要提权，看看suid提权无果，为什么是nc？看一看端口的连接情况。

发现一个9999端口挺奇怪的，nc连一下看看应该是提权的入口，况且没有显示进程说明很大可能是root开的。

在ctf用户的目录下发现main文件，反编译分析就是需要pwn的文件。

场景6、7

根据提示访问8080端口，发现是Shiro框架写的，抓个登录的包，发现deleteMe，工具嗦一下反序列化。

执行命令发现该机子的IP是192.168.0.3，应该是将服务转发到了外网机子的8080端口上。接下来找flag。

提交flag错误，场景6的flag应该在网站里面，源代码也没有，御剑扫一下，原来在robots.txt里面。

场景8

接下来是提权拿root的flag，尝试suid提权。

现在VPS上反弹一个shell，记得VPS要开放端口。

小记 SUID find提权_Crispr-bupt的博客-CSDN博客

场景9

VPS的终端看得太难受了，用frp穿透一下在kali里面拿一个终端。提示内网，fscan扫一下。

Releases · shadow1ng/fscan · GitHub

可以发现还有一个192.168.0.4的主机没有访问到，看title是一个博客。用Neo-reGeorg做一个代理。

GitHub - L-codes/Neo-reGeorg

成功访问。

通过报错我们拿到了CMS：ThinkPHP V5.0.20

直接上工具嗦，发现RCE，蚁剑拿下。

场景10

根据提示在数据库里面，直接找配置文件。

发现蚁剑直接连不了，所以用frp做了一个端口的转发送到VPS上。

场景11

尝试了一下本地漏洞扫描的脚本，扫出来的CVE也没能成功利用，直接看WP吧。

回头看来找suid的时候发现了/usr/bin/pkexec，这是不是也是一种提示呢。
