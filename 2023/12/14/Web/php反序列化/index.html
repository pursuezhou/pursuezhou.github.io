魔术方法

  • __construct()：构造函数，在实例化一个对象的时候调用，序列化和反序列化不会调用。

  • __destruct()：析构函数，在对象的所用引用被删除或者当对象被显示销毁时执行，序列化不会调用，反序列化得到的对象销毁时调用。

  • __sleep()：序列化之前被调用，返回一个包含对象中所有应被序列化的变量名称的数组，删除不必要的属性。

  • __serialize()：当和sleep同时存在时忽略sleep方法。

  • __wakeup()：反序列化之前被调用，预先准备对象需要的资源，返回void。

  • __unserialize()：当和wakeup同时存在时忽略wakeup方法。

  • __tostring()：把对象被当成字符串使用时调用。

  • __invoke()：把对象被当成函数使用时调用。

  • __call()：当调用一个不存在的方法，传入不存在方法的名称和参数数组。

  • __callStatic()：静态调用或调用成员常量时使用的方法不存在，传入不存在方法的名称和参数数组。

  • __get()：调用的成员属性不存在或访问私有变量，传入不存在成员属性的名称。

  • __set()：给不存在或不可访问的成员属性赋值，传入不存在成员属性的名称和参数数组。

  • __isset()：对不存在或不可访问属性使用 isset 或 empty，传入不存在成员属性的名称。

  • __unset()：对不存在或不可访问属性使用，传入不存在成员属性的名称。

  • __clone()：当使用clone拷贝完对象后调用。

    进行回溯，首先理解 Laravel 服务容器一个特性。在 PHP 中，类本身通常不能直接当作数组使用。但在 Laravel 框架的上下文中，当你看到像 $this->app[Kernel::class] 这样的代码，它其实是在利用 PHP 的数组访问语法来从服务容器中解析服务。这是 Laravel 服务容器的一个特性，而不是标准 PHP 的行为。下面解释这个机制：

    1. 服务容器（Service Container）：
      • Laravel 的服务容器是一个用于依赖注入的强大工具。它允许你将各种服务（如类实例）绑定到容器中，并在需要时从容器中解析出这些服务。
    2. 数组访问接口（ArrayAccess Interface）：
      • Laravel 的服务容器实现了 PHP 的 ArrayAccess 接口。这个接口允许对象像数组那样被访问。
      • 实现 ArrayAccess 接口意味着你可以使用数组语法来获取（offsetGet）、设置（offsetSet）、检查（offsetExists）和移除（offsetUnset）对象的元素。
    3. 在 Laravel 中使用数组语法：
      • 当你使用像 $this->app[Kernel::class] 这样的代码时，你实际上是在调用服务容器的 offsetGet 方法。Kernel::class 作为一个键，传递给这个方法，返回对应的服务实例。
      • 这种语法提供了一种便捷的方式来从容器中检索服务，而不是使用更长的方法调用（例如 $this->app->make(Kernel::class)）。
